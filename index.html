Semantic Search Application
Billion-Scale Vector Search over Enterprise Data

Inspired by Uber's billion-scale vector search architecture, I designed and built an end-to-end semantic search platform that enables natural language querying over large-scale structured datasets. The system replaces brittle keyword search with embedding-based retrieval, surfacing semantically relevant results even when exact terms don't match.

Key Technical Decisions
  • Distributed preprocessing and batch embedding generation via PySpark for production-grade scale
  • Chose OpenSearch k-NN plugin over dedicated vector databases for operational simplicity in enterprise environments

Streaming Series Breakout Detection
Multi-Signal Momentum Scoring for Streaming Shows

Detects when TV shows "break out" on streaming platforms by combining four public data signals — Reddit discussion, Wikipedia pageview spikes, YouTube trailer engagement, and TMDB popularity — into a composite Momentum Score. A show scoring above 75 is flagged as a breakout candidate with an AI-generated reason explaining why it's trending.

Key Technical Decisions
  • Built four Fivetran custom connectors (Python SDK) for Reddit, TMDB, Wikipedia, and YouTube — including delta detection via state tracking to skip already-ingested records
  • Chose Snowflake Cortex SENTIMENT() for Reddit comment analysis — zero infrastructure overhead, runs natively inside the warehouse
  • dbt Bronze → Silver → Gold medallion architecture keeps raw ingestion, transformation, and business logic cleanly separated
  • Wikipedia 7-day rolling average + day-over-day delta catches both sustained surges and sudden spikes in a single metric
  • Breakout threshold of 75/100 with a Cortex LLM-generated MOMENTUM_REASON field explaining new episodes, legacy buzz, or pre-release hype
